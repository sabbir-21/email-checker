Privacy Policy for Gmail Email Verifier Extension

Last updated: 11/26/2025

Thank you for using Gmail Email Verifier, a browser extension developed by Sabbir Ahmed.
This privacy policy explains how the extension handles your data.
We value your privacy and designed this tool with data minimization and user safety as first priorities.

1. Overview

Gmail Email Verifier is a security-enhancement extension that helps you:

Identify suspicious or spoofed emails

Inspect sender domains and MX records

Detect phishing indicators

Detect dangerous attachments

Analyze reply-to mismatch

Inspect metadata available in Gmail

Warn about potentially harmful email content

The extension does NOT collect, store, upload, or transmit any personal data.

2. Information We Do NOT Collect

The extension does NOT collect, store, or transmit:

Your emails

Your Gmail content

Sender or recipient information

Attachments

Personal data

Browser history

IP address

Cookies

Any identifiers

No logs are created.
No analytics services are used.
No third-party tracking occurs.

3. What the Extension Actually Reads

The extension only reads what is already visible on the Gmail page to perform security checks, including:

The displayed sender email

The displayed reply-to header

The displayed attachments' file names

The visible metadata (mailed-by, signed-by, etc.)

This information is processed locally on your device and never leaves your browser.

4. What External Requests Are Made

The extension may make requests to:

Google DNS (dns.google)

Used only for checking MX records of sender domains.

(Optional) WHOIS API (if enabled by the user in the code)

Used only to retrieve the domain creation date.

No personal data is ever transmitted.
Only the domain name part of the sender’s email (e.g., example.com) is used.

5. No Data Storage

The extension:

Does NOT store any data in remote servers

Does NOT use databases

Does NOT save email information locally

Does NOT upload anything to a backend server

All checks happen in real time, in your browser, in memory.

6. No Account Required

The extension does not require:

Signup

Login

API keys (unless the user manually enables optional WHOIS functionality)

Any personal information

7. Permissions Explanation
https://mail.google.com/*

Needed to read visible email metadata inside Gmail for analysis.

dns.google

Used only to check MX records.
We do not transmit user data — only the sender’s domain.

storage (if used)

For storing basic extension settings, not user data.
(You may delete this if unused.)

8. No Third-Party Sharing

We do NOT sell, share, or transmit any user information to:

Third-party companies

Advertisers

Analytics providers

Governments

Because we do not collect any user data in the first place.

9. Data Security

Since no data is stored or transmitted, risks of exposure are minimal.
All processing happens client-side.

10. Children's Privacy

The extension does not target children and does not collect personal information from anyone.
